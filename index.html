Problem

Tunisian olive oil mills run almost entirely on paper: manual weight logs per customer batch, handwritten invoices, zero visibility into energy costs per cycle. Mill owners had no way to correlate input parameters — olive quantity, moisture %, press temperature, cycle count — with energy consumption. The goal: digitalize the entire workflow and give owners an ML-backed tool to forecast energy spend before starting a batch.

Architecture
Flutter Mobile → Django REST API → PostgreSQL → scikit-learn Endpoint → Energy Prediction

The ML module is a separate Python service integrated into Django via an internal prediction endpoint. Data flows: weight entry on mobile → Django validates + persists → invoice generated → batch parameters optionally sent to prediction endpoint for energy cost estimation.

What It Does
Automated Invoicing

Invoices generated from batch records using oil yield formulas and configurable pricing. No manual calculation.

Digital Weight Tracking

Replaces paper logs. Each customer batch is stored with timestamp, weight, and operator ID.

ML Energy Prediction

Given batch parameters, the API returns predicted energy cost. Trained on real mill data — R²=0.95 on test set.

Customer Records

Full customer history across seasons: total weight delivered, outstanding balances, batch count.
